ETL Pipeline Using Apache Airflow for Real Estate Data Analysis

Problem Statement: Real estate data is often scattered, unstructured, and manually processed, making timely market analysis inefficient. There is a need for an automated ETL (Extract, Transform, Load) pipeline to clean, transform, and store real-estate listings data for analytics and visualization.

Technologies Used: Apache Airflow, Python, Pandas, AWS S3, CSV, Docker, Jupyter Notebook, Crontab (for scheduling in production)

Duration: 2 weeks

Workflow:

  1. Extract real estate listing data from external sources (web, CSVs, APIs).
  2. Transform using Pandas: clean nulls, format dates, and calculate derived metrics like price per square foot.
  3. Load into CSV and AWS S3 bucket for downstream consumption.
  4. Schedule the pipeline using Airflow DAGs to run daily.

YouTube Transcript Summarizer Using NLP

Problem Statement: Users often don’t have time to watch lengthy YouTube videos but still want to understand the core message. There’s a need for an automated tool that fetches YouTube video transcripts and summarizes them using NLP techniques.

Technologies Used: Python, Hugging Face Transformers, YouTube Transcript API, Streamlit, Spacy, NLTK, Flask, GitHub Pages (for deployment)

Duration: 3 weeks

Workflow:

  1. Extract transcript using YouTube Transcript API.
  2. Preprocess text (tokenization, stopword removal).
  3. Apply NLP summarization using transformer-based models (BART/T5).
  4. Display output in a Streamlit web app.
