ETL Pipeline Using Apache Airflow for Real Estate Data Analysis

Problem Statement:
Real estate data is often scattered, unstructured, and manually processed, making timely market analysis inefficient. There is a need for an automated ETL (Extract, Transform, Load) pipeline to clean, transform, and store real-estate listings data for analytics and visualization.

Technologies Used:
Apache Airflow, Python, Pandas, AWS S3, CSV, Docker, Jupyter Notebook, Crontab (for scheduling in production)

Duration:
2 weeks

Workflow:

  1. Extract real estate listing data from external sources (web, CSVs, APIs).
  2. Transform using Pandas: clean nulls, format dates, and calculate derived metrics like price per square foot.
  3. Load into CSV and AWS S3 bucket for downstream consumption.
  4. Schedule the pipeline using Airflow DAGs to run daily.

Description:
The project involves building a robust and scalable ETL pipeline using Apache Airflow for real estate data processing and analysis. The objective was to automate the flow of data from raw input to a cleaned, analysis-ready format. This pipeline is designed to extract raw real estate data from CSV files or APIs, transform it to a usable structure, and store it in both local CSV format and cloud (AWS S3).

Airflow DAGs were configured to orchestrate the pipeline and handle failures. Python and Pandas were used to implement the transformation logic which includes removing missing entries, converting date formats, and calculating new metrics such as price_per_sqft. The processed data is saved locally for quick analysis and also pushed to AWS S3 for long-term storage and integration with BI tools.

This project showcases how Airflow, when combined with lightweight tools like Pandas, Docker, and AWS, can handle the automation of complex data workflows in real estate or similar domains. The pipeline is modular and production-ready, with scope to expand into database storage or ML-based price prediction models.

YouTube Transcript Summarizer Using NLP

Problem Statement:
Users often don’t have time to watch lengthy YouTube videos but still want to understand the core message. There’s a need for an automated tool that fetches YouTube video transcripts and summarizes them using NLP techniques.

Technologies Used:
Python, Hugging Face Transformers, YouTube Transcript API, Streamlit, Spacy, NLTK, Flask, GitHub Pages (for deployment)

Duration:
3 weeks

Workflow:

  1. Extract transcript using YouTube Transcript API.
  2. Preprocess text (tokenization, stopword removal).
  3. Apply NLP summarization using transformer-based models (BART/T5).
  4. Display output in a Streamlit web app.
